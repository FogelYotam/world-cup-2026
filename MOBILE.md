## 🤖 הוראות לסוכן (Claude) — בצע לפי הסדר

**אל תשתמש ב-Gemini. קרא את הצילומים בראייה שלך** (Gemini טועה ב-RTL/אוריינטציה). הליבה רצה בלי תלויות — אין צורך ב-`pip install`.

1. **`git pull --rebase origin main`** — לקבל את העדכני (חובה לפני כל עבודה).

2. **זהה מה צורף:**
   - **צילומי ניחושים מ-KICKOFF** (כרטיסים עם "ניחשת X-Y" + badge נקודות: בול=+3, כיוון=+1, פספוס=0, קנס חמור=−1). הקבוצה הראשונה/ימנית = בית. **התעלם מצילומי קלט/עריכה** (תיבות ניקוד, בלי badge).
   - **דוחות של התוכנה** (HTML/טלגרם, "המלצה לניקוד X-Y" / "תוצאה מומלצת X-Y" לכל משחק) — אלה ניחושי המודל **לפני המשחק**.

3. **חלץ לכל משחק:** `home`, `away`, `user_home`, `user_away` (מהצילומים שלך), ואם יש דוחות — גם `model_home`, `model_away` (מהדוח, מיושר ל-home שלך).

4. **בדיקת אוריינטציה (חובה):** אם `predictor.match_points(user, actual)` ≠ ה-badge שבכרטיס → קראת הפוך; החלף.

5. **הרץ את הכלי** (שומר, מיישב מול `data/db.json`, ומחזיר "אתה מול המערכת"):
   ```python
   import kickoff_predictions as kp
   print(kp.process([
       {"home": "...", "away": "...", "user_home": X, "user_away": Y,
        "model_home": A, "model_away": B},   # model_* רק אם יש דוח
       ...
   ]))
   ```
   ניקוד KICKOFF: מדויק +3 · כיוון +1 · הפוך −1 · פספוס 0 · **בלי בונוס שערים**.

6. **אם אין דוחות** — `process` יחשב את המודל לבד, אבל לעבר זה **in-sample/מנופח**. ציין זאת, והעדף תמיד את ניחוש המודל **מהדוח**.

7. **שמור:** `git add data/my_predictions.json && git commit -m "..." && git push`.

8. **החזר למשתמש:** את טבלת "אתה מול המערכת" (ניקוד + מדויק/כיוון), ואם ביקש — פירוט משחק-משחק.

### בקשת "הצג ניקוד נוכחי" (בלי תמונות)
`git pull` ואז `python kickoff_predictions.py` (ללא ארגומנט — מדפיס את הסיכום המצטבר), או `predictions_log.format_summary_he()`. לפירוט משחק-משחק — חשב `match_points` לכל רשומה מוכרעת ב-`data/my_predictions.json`.
