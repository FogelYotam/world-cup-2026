## 🏆 זרימה A — חוות-דעת על ההרכב (צילום קבוצת פנטזי)

1. **`git pull`**.
2. **קרא את הצילום** (ראייה שלך): 15 השחקנים, מי ב**הרכב הפותח** מול **ספסל**, המערך, הקפטן (C) והסגן (VC), והיריבה שליד כל שחקן.
3. **עדכן את `data/my_team.json`** כך שישקף את הקבוצה — `squad` (שם+נבחרת+עמדה), `captain`, `vice_captain`. (זה מה שמזין את התראות ההרכב.)
4. **שלוף הקשר לכל שחקן** (מ-db.json, ללא רשת):
   ```python
   import md_fantasy, json
   print(json.dumps(md_fantasy.squad_context(), ensure_ascii=False, indent=1))
   ```
   מחזיר לכל שחקן: `opponent`, `ease`, `clean_sheet_pct`, `recent_points`, `ownership`, `scouting_bonus`, `availability`.
5. **תן חוות-דעת** לפי הנתונים:
   - 👑 **קפטן/סגן** — האם בחר תקרה גבוהה עם משחק קל (חתוך לפי `recent_points` × `clean_sheet_pct`).
   - 🪑 **כוכב על הספסל בטעות** — אם שחקן-על (נקודות/תקרה גבוהות, משחק קל) על הספסל בזמן שמתחיל חלש יותר פותח → סמן והמלץ להעלות (זכור: max 3 חלוצים, מערך חוקי).
   - 🧤 **שער-נקי** — מי מהמגנים/שוערים במשחק עם `clean_sheet_pct` גבוה; דאבל מאותה נבחרת חזקה = בונוס.
   - ⚠️ **משחק קשה / זמינות** — סמן מתחילים עם `ease` נמוך או `availability` שאינו "זמין".
   - 💎 **שדרוגים** — הצע מ-`db['top_picks']` (כוכבים) ו-`db['differentials']` (⭐ scouting bonus <5%).
6. **רוטציה יומית** (אם רלוונטי) — המשחקים פרושים על כמה ימים; אפשר חילופים בסיום כל יום. בנה תוכנית "מי משחק כל יום + קפטן יומי". **שים לב:** חילוף ידני מבטל את ה-auto-subs לכל הסבב.
7. **שמור:** `git add data/my_team.json && git commit -m "update squad from screenshot" && git push`.

> סגל מומלץ מאפס למחזור: `python md_fantasy.py`.

## 📊 זרימה B — ניחושים (אתה מול המערכת)

1. **`git pull`**.
2. **זהה מה צורף:**
   - **צילומי KICKOFF** — כרטיסים עם "ניחשת X-Y" + badge: בול=+3, כיוון=+1, פספוס=0, קנס=−1 (רק הפך מדויק/מראה). הקבוצה הראשונה/ימנית = בית. **התעלם מצילומי קלט/עריכה** (בלי badge).
   - **דוחות התוכנה** (טלגרם/HTML, "המלצה/תוצאה X-Y") — ניחוש המודל **לפני המשחק**.
3. **חלץ לכל משחק:** `home`, `away`, `user_home`, `user_away`, ואם יש דוח — גם `model_home`, `model_away` (מיושר ל-home שלך).
4. **בדיקת אוריינטציה (חובה):** אם `predictor.match_points(user, actual)` ≠ ה-badge → קראת הפוך; החלף.
5. **הרץ:**
   ```python
   import kickoff_predictions as kp
   print(kp.process([{"home": "...", "away": "...", "user_home": X, "user_away": Y,
                      "model_home": A, "model_away": B}, ...]))   # model_* רק אם יש דוח
   ```
   ניקוד KICKOFF: מדויק +3 · כיוון +1 · הפך-מדויק −1 · פספוס 0 · **בלי בונוס שערים**.
6. **בלי דוחות** — `process` **מנסה קודם את ניחוש-הדוח השמור** (`data/report_predictions.json`, נכתב אוטומטית בכל דוח יומי — הוגן/טרום-משחק). רק אם אין שם, הוא מחשב מחדש (in-sample/מנופח; ציין זאת). אם המשתמש מצרף דוח HTML חדש — אפשר להוסיף אותו לארכיב ב-`data/reports/` ולחלץ ממנו ל-store.
7. **שמור:** `git add data/my_predictions.json && git commit && git push`.
8. **החזר** טבלת "אתה מול המערכת". בלי תמונות: `python kickoff_predictions.py` (סיכום מצטבר).
